Maven

记录maven的一些操作，设置，当然免不了还有坑

首先 maven能 解决的问题

  1. 解决jar包的冲突，控制版本，手动引入
  2. 编译代码，单元测试
  3. 部署项目

依赖管理，maven工程对 jar包的管理过程：把jar包放仓库里面，pom文件里面是jar包的坐标
一键：maven tomcat:run

maven的配置

  1. 配置环境变量
    在系统变量里面新建变量
    MAVEN_HOME
路径
    path
    %MAVEN_HOME%\bin
    怎么检查：maven -v 
    Default: ${user.home}/.m2/repository  这个是默认仓库的地址
<localRepository>/path/to/local/repo</localRepository> 这个是设置本地仓库的命令
<localRepository>D:\Maven\MyJar</localRepository> 例如，设置里面都是注释，要重新设置在注释外面
    公司里面叫私服或者远程仓库，联网的叫中央仓库
    1. maven 目录结构，也是规范
    三大部分 ：核心代码文件，配置文件，测试代码部分
    src/main/java 核心代码
src/main/resources 配置文件
src/test/java  测试代码
src/test/resources 测试配置文件
src/main/webapp  页面资源，js，cs，图片等资源
    1. maven的常用命令
      maven clean 这个执行了以后，当前目录下的target目录就会清除掉，target里面是写编译后的Java文件，就是会删掉项目中的编译后文件
      接手别人的项目后，就用这条命令删除本地的编译文件
      maven compile 编译该项目，生成本地target文件
      maven test 生成编译的测试文件，该命令还会同时生成target文件
      maven package 打包命令
      maven install 把项目的jar包安装在本地仓库
    2. maven 一键构建
      编译–测试–打包–安装–发布 这是一套生命周期：默认设生命周期 还有站点生命周期
      pom里面有插件，依赖和运行环境信息
    3. Idea集成本地maven
      在maven设置里面设置本地maven目录，配置文件和仓库路径
      runner 里面配置-DarchetypeCatalog=internal 本地有就找本地，不用去中央仓库找
